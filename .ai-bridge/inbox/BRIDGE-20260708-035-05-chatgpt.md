## Defect A - runtime getter precedence

Current omitted-profile behavior:

1. calls `Game.Data.getUiProfile()` when the getter exists;
2. if the getter returns an unrecognized value, continues to `Game.Data.UI_PROFILE`;
3. therefore a stale raw `UI_PROFILE = "alpha"` or `"boomer"` can override an unrecognized getter result.

Frozen required behavior:

- explicit non-empty recognized input -> exact profile;
- explicit non-empty unknown input -> `default` immediately;
- omitted, null or blank input:
  - if `Game.Data.getUiProfile` exists, call it exactly once and use only that result;
  - otherwise read `Game.Data.UI_PROFILE`;
  - recognized runtime value -> exact profile;
  - unrecognized runtime value -> `default`;
- never consult `Game.Data.UI_PROFILE` when a getter exists, regardless of the getter result.
