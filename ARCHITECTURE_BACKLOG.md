### Краткое описание

Economy Canon v1.1 включает 4 системных изменения экономики и механики конфликтов:

1. **Тон как давление**  
   Победа игрока на высоком тоне над более слабым оппонентом списывает Influence у победителя и ограничивает прирост REP (обнуление или минимальный плюс).

2. **VL5 "послать"**  
   Кнопка "послать" (уже существует, доступна с VL5) немедленно закрывает конфликт без боя и всегда даёт штраф REP с явным reason и battleId. Points не меняются.

3. **Реванш**  
   Проигравший может запросить реванш, оплачивает Points напрямую оппоненту (трансфер игрок→игрок). Оппонент может принять (создать новый баттл) или отклонить без штрафов. Реваншей может быть сколько угодно, цена растёт при повторных запросах.

4. **NPC токсик/бандит/мафиози непобедимы**  
   Для обычного игрока токсик, бандит, мафиози непобедимы. Победа возможна только спецролью (коп/главкоп). Обычный игрок может только проиграть или выйти (escape/dismiss) с соответствующими последствиями.

### Инварианты (границы внедрения)

**Что входит:**
- Изменения механики экономики (REP, Influence, Points) в рамках battle_end, dismiss(VL5), revenge
- Правила NPC-конфликтов (победа/поражение/escape для обычного игрока vs спецроль)
- Новые причины (reasons) для трансферов REP и Points с обязательным battleId/eventId
- Рост цены реванша при повторных запросах

**Что НЕ входит / Запреты:**
- Эмиссия Points (Points только трансфер игрок↔игрок, без создания "из воздуха")
- Изменение базовой механики аргументов (аргументы остаются единственным инструментом боя)
- Создание новых кнопок UI (кнопка "послать" уже существует)
- Нарушение инвариантов REP v1.0 по логированию (reason, battleId/eventId, moneyLog, moneyLogByBattle)

**Зафиксированные инварианты:**
- Points конечны, старт 10 на игрока
- Любые изменения Points только трансфер игрок-игрок, без эмиссии
- REP эмитится и сгорает, не является конечным ресурсом
- Influence изнашивается
- Аргументы остаются единственным инструментом боя
- Кнопка "послать" уже существует и доступна с VL5
- Инварианты REP v1.0 по логированию сохраняются (reason, battleId/eventId, moneyLog)

### Риски (зафиксированные для проработки)

- Эксплойт реванша как фарм Points у сильного игрока
- Неверное определение "сильнее/слабее" по силе
- Регресс существующих экономических хуков
- Несогласованность с текущим gate REP v1.0

### Тест-сценарии (минимальный набор для будущего PASS)

- dismiss(VL5)
- revenge accept/decline с ростом цены
- взаимодействие с токсик/бандит/мафиози (обычный игрок vs спецроль)
