# Economy wave 3 — UI gate package (rematch notifications/actions)

Цель: определить, какие UI/UX изменения допустимы для поддержки механики реванша (wave 3), без “маркетинговых” обещаний и без изменения механики.

## 1) Предпосылка
Wave 3 core (см. `ECONOMY_WAVE3_SCOPE.md`) реализует request/accept/decline без UI.
Этот пакет — только про UI слой: как игрок запускает и видит реванш.

## 2) Разрешённые изменения UI (предложение)
Только UI слой:
- Показ уведомления в событиях/баттле/ЛС: “`<name>` просит реванш”
- Две кнопки/действия у адресата: `Принять` / `Отклонить`
- Показ результата: “Реванш принят” / “Реванш отклонён”

Требование: UI **не обещает** игроку экономические дельты (Points/REP/Influence) числами. Никаких “-1 💰”, “REP -”, и т.п.

## 3) Запрещено
- Любые изменения экономической механики в UI
- Любые числовые обещания/дельты в UI
- Любые новые кнопки/смыслы, не требуемые реваншем

## 4) Затрагиваемые файлы (ориентир)
Только UI:
- `AsyncScene/Web/ui/ui-battles.js` (кнопки/карточка баттла)
- `AsyncScene/Web/ui/ui-events.js` или `AsyncScene/Web/ui/ui-chat.js` (уведомления)
- `AsyncScene/Web/ui/ui-dm.js` (если реванш пойдёт через ЛС)

## 5) PASS/FAIL критерии (gate + аудит)
### PASS
- UI даёт возможность вызвать request/accept/decline и показывает нейтральные уведомления
- UI не содержит числовых обещаний/дельт по экономике
- Изменены только UI файлы

### FAIL
- UI содержит “цены/награды/штрафы/дельты” числами или эмодзи валют
- UI меняет механику/состояние напрямую, минуя core API
