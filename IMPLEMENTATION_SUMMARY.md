# IMPLEMENTATION SUMMARY - AsyncScene REP Economy & UI Fixes

## Изменённые файлы

### 1. `AsyncScene/Web/conflict/conflict-economy.js`
**Задача A**: REP за исход баттла по разнице сил Δ (тон, не influence)
- Добавлена функция `getToneDelta(battle)` для вычисления разницы сил по тону (y=1, o=2, r=3, k=4)
- В режиме circulation (строки ~569-577):
  - Победа: Δ≥2 → 0 REP, |Δ|≤1 → +1 REP, Δ≤-2 → +2 REP
  - Поражение: Δ≤-2 → +1 REP, |Δ|≤1 → -1 REP, Δ≥2 → -2 REP
- В legacy режиме (строки ~606-611, ~724-735):
  - Та же логика, но с использованием старой инфраструктуры
- REP начисляется через `transferRep` с reason `rep_battle_win_delta` / `rep_battle_lose_delta` и battleId

**Задача F**: Убрать тост "Победа!"
- Закомментирован `Game.UI.pushSystem("🏆 Победа!")` в legacy режиме (строка ~561)
- Остались только тосты дельт (REP, пойнты)

### 2. `AsyncScene/Web/events.js`
**Задача B**: Пойнт-конвертер в REP при голосовании
- В `helpEvent` (строки ~869-900):
  - После успешного голосования сразу начисляется +1 REP (участие) через `transferRep`
  - Показываются тосты: "-1 💰 (голосование)" и "+1 REP (участие)"
- В `payoutCrowdPool` (строки ~131-169):
  - Убрано дублирование начисления REP за участие (теперь только при клике)
  - При завершении: majority → +2 REP, minority → -2 REP
  - Тосты результата: "+2 REP (большинство)" + "+1 💰 (возврат)" или "-2 REP (меньшинство)"

## Инварианты

- **REP начисляется по Δ тонов**, не influence
- **Crowd голосование**: -1p на клике, +1 REP участие сразу, ±2 REP результат при завершении
- **Реванш**: только пойнты, REP не трогает
- **Типы аргументов**: проверка по группе (ABOUT/WHO/WHERE/YN), не по конкретному сабтиру
- **Тосты**: без "Победа!", только дельты
- **Коп**: говорит от первого лица, DM при виктимизации
